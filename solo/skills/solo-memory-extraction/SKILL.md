# 长效事实提取原则

用户的日常记录中经常隐含**持久性个人事实**——这些信息不是"今天发生了什么"的事件，而是"用户是什么样的人"的背景。识别并持久化这些事实是高质量记录的核心职责之一。

## 判断标准

这条信息是否具有跨会话价值？
问自己：**如果未来某次对话中不知道这个事实，会不会导致理解偏差或建议失当？** 如果是，它值得写入长效记忆。

## 典型场景（理解原则，不要机械匹配）

- 用户描述某种身体反应或症状，从中可推断出用户存在某种**持续性健康状况**
- 用户提到某人的身份或关系，这是**首次出现**的人物信息
- 用户透露居住环境、通勤方式、工作性质等**稳定的生活结构**
- 用户表达对某事物的长期态度、恐惧、偏好（非临时情绪）
- 用户提及某种技能、经历、资质等**个人能力画像**
- 用户陈述**职业履历**（入职时间、工龄、岗位变动）或**人生里程碑**
- 用户在**纠错或补充说明**中澄清了关于自己的事实（如"我其实已经工作六年了"）——纠错本身说明该事实对后续交互很重要，必须入库

## 操作要求

1. 遇到上述信号时，在调用 solo_record 的**同一轮**，同时调用 solo_remember 将持久性事实写入长效记忆
2. 如果用户陈述的是纯个人事实（不涉及具体事件），**直接调用 solo_remember**，无需先调 solo_record
3. solo_remember 的 content 应提炼为**事实陈述**（如"用户患有过敏性鼻炎"），而非复制原始消息
4. 如果记忆文件中已有相关条目（如已有 medical_history.md），将新发现追加或更新到该条目中
5. 只对**首次出现或发生变更**的事实调用 solo_remember；日常重复事件不需要
6. 事件本身的记录（何时发作、症状细节等）仍然通过 solo_record 完成，solo_remember 只存稳定的背景事实

## 不提取的情况

- 信息完全是一次性事件，不具备跨会话参考价值
- 信息已经存在于已有的 memory 文件中且没有变化
- 信息过于模糊，无法提炼为可靠的事实陈述
